Magnitude EK - Analysis

Introduction

The Magnitude Exploit Kit is a tool used by cybercriminals to exploit vulnerabilities in victims' systems. FortiGuard Labs has discovered a variant of this kit that exploits a specific technique with VBScript to load the .NET assembly directly from memory.

Its operation is based on a series of carefully orchestrated steps to ensure the efficient delivery of its malicious payload. Here is a detailed explanation of these steps:

  1. Redirection via advertising network (Ad network 302 redirection)
    • What is it?: When a user visits a website, they may be redirected to another site or domain without their knowledge. This redirection is often caused by malicious advertising networks that use an HTTP 302 redirection to direct the user to malicious content.
    • Why is it used?: Cybercriminals exploit these networks to target a large number of users. By using a redirection, they can mask the malicious origin and deceive security mechanisms that could block a malicious site directly.
  1. Magnitude EK Portal containing obfuscated JS
    • What is it?: Once the user is redirected, they land on a page controlled by Magnitude EK. This page contains JavaScript (JS) that is obfuscated, meaning its code is made unreadable or hidden to avoid detection.
    • Why is it used?: Obfuscation is a common technique used to hide the true purpose of the code. In this case, the obfuscated JS is likely used to exploit a vulnerability in the user's browser or to download an additional payload.
  1. Redirection to a second domain with VBScript exploitation (CVE-2018-8174) and .NET payload
    • What is it?: After executing the obfuscated JS, the user is again redirected, this time to a second domain. On this page, VBScript-based exploitation is used, specifically exploiting the CVE-2018-8174 vulnerability. In addition, a .NET payload is also delivered to the victim's machine.
    • Why is it used?: CVE-2018-8174 is a known vulnerability that allows remote code execution. By exploiting this vulnerability, attackers can execute malicious code on the victim's machine. The .NET payload is likely malware or ransomware that is executed once the vulnerability is exploited.

    Magnitude and GandCrab Ransomware

    • Propagation of GandCrab: GandCrab ransomware is spread via malspam, social engineering schemes, and exploit kit campaigns. Magnitude EK, previously loyal to its own Magniber ransomware, is now being used to spread GandCrab.
    • Fileless technique: Magnitude is now using a fileless technique to load the ransomware payload, making its detection more difficult.
    • Injection into explorer.exe: Once the payload is injected into explorer.exe, it immediately attempts to restart the machine. By suspending this process, it is possible to extract the GandCrab DLL from memory.
    • GandCrab ransom note: After a successful infection, files are encrypted with the .CRAB extension and a ransom note is left with instructions.
