Poweliks - Analysis

Introduction

Poweliks, first identified in 2014, marked a significant evolution in the malware landscape. Unlike most malware that resides on a computer's hard drive, Poweliks is entirely "fileless", operating from the computer's memory and registry. This unique characteristic makes it particularly insidious and difficult to detect with traditional methods.

Technical Characteristics of Poweliks

  1. Fileless Nature: Poweliks does not rely on a traditional executable file. Instead, it uses the Windows registry to store its malicious code, particularly using hidden registry keys.
  1. Code Injection: Rather than functioning as an independent process, Poweliks injects code into legitimate processes, particularly Internet Explorer. This allows it to hide its malicious activity behind seemingly legitimate processes.
  1. Persistence: To ensure persistence after a reboot, Poweliks places code in hidden registry keys. When the computer is restarted, the code is executed, allowing Poweliks to resume its malicious activity.
  1. Evasion: One of the most notable evasion techniques used by Poweliks is the creation of registry entries using a NULL pointer as the name. This makes these entries invisible to the standard Windows registry editor.

Operating Mechanisms

  1. Initialization: Upon its initial execution, Poweliks creates two registry entries. The first contains the startup code, while the second contains an encoded version of the second stage of infection.
  1. Execution of Startup Code: The startup code is essentially a JavaScript instruction that reads the registry entry containing the second stage of infection, decodes it, and executes it.
  1. Use of Rundll32: Poweliks executes the JavaScript code using the rundll32 interface, a method described in detail by Stormshield.
  1. Execution of Payload: The second stage of execution uses PowerShell to create the dllhost.exe process, then injects and executes the actual payload.

Detection and Countermeasures

  1. Memory Monitoring: Given that Poweliks primarily resides in memory, constant monitoring of memory is essential to detect any suspicious activity.
  1. Registry Monitoring: Registry modifications, particularly hidden keys or keys with non-standard names, may indicate the presence of Poweliks.
  1. Heuristic Analysis: Heuristic methods, which detect malicious behaviors rather than specific signatures, can be particularly effective against Poweliks.
  1. Isolation and Analysis: If suspicious activity is detected, it must be isolated to prevent further propagation or damage. A thorough analysis can then be performed to determine the exact nature of the threat.

Conclusion

Poweliks represents a class of malware that defies traditional detection and removal methods. Its fileless nature, combined with its ability to hide behind legitimate processes, makes it particularly insidious. However, with a deep understanding of its operation and advanced detection methods, it is possible to protect systems against this threat.
