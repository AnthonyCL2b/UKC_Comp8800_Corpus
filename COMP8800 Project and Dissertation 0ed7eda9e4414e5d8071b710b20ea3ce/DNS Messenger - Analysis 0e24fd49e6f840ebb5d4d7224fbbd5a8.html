DNS Messenger - Analysis

Technical Characteristics of DNSMessenger

  1. Fileless Nature:
    • Memory Execution: DNSMessenger executes its scripts directly in memory, avoiding leaving traces on the victim's hard drive.
    • Use of Powershell: The malware uses Powershell, a tool built into Windows, to execute its scripts. This allows it to blend in with legitimate traffic and avoid detection.
    • No Executable File: Unlike traditional malware, DNSMessenger does not require an executable file to function. Everything is done through scripts and commands.
  1. Use of DNS as C2 Channel:
    • DNS TXT Queries: DNSMessenger uses DNS TXT queries to communicate with its command-and-control (C2) server. These queries are often less monitored than other types of traffic, allowing the malware to go unnoticed.
    • Data Exfiltration: Stolen data is transmitted to the C2 server via these DNS queries, thus masking the exfiltration.

Operating Mechanisms

  1. Malicious Word Document:
    • Infection begins with a malicious Word document sent via a phishing email.
    • This document contains macros that, when activated, trigger the malicious Powershell script.
  1. Multi-Step Powershell Scripts:
    • Step 1: The initial script is triggered by the Word document. It decodes and decompresses an obfuscated Powershell script and launches it.
    • Step 2: This script prepares the environment for the next step, including setting up functions and establishing parameters for C2 communication.
    • Step 3: This script establishes communication with the C2 server via DNS queries and waits for commands.
    • Step 4: This is the actual command and control script. It receives commands from the C2 server, executes them, and returns results via DNS.
  1. Evasion and Persistence:
    • Obfuscation: The scripts are obfuscated to make analysis more difficult.
    • Windows Registry: DNSMessenger uses specific registry keys to ensure persistence on the infected system.
    • Scheduled Tasks: It also creates scheduled tasks to ensure it is run regularly.

Countermeasures and Detection

  1. DNS Traffic Monitoring: Since DNSMessenger uses DNS for communication, constant monitoring of DNS traffic is essential.
  1. Heuristic Analysis: Rather than looking for specific signatures, a heuristic approach detects malicious behavior.
  1. Macro Blocking: Preventing automatic execution of macros in Word documents can stop initial infection.
  1. Advanced Security Solutions: Tools like Cisco Umbrella can help detect and block malicious DNS queries.

Conclusion

DNSMessenger is an example of the sophistication of current threats. Its fileless nature and use of DNS as a C2 channel make it particularly insidious. Effective defense requires a combination of proactive monitoring, advanced security tools, and ongoing user education to avoid traps like phishing emails.
